# Full QA Workflow

**使用方法**: `/full-qa` を実行することで、プロジェクト全体の統合的なQAを進行します。

## Step 1: MainActivity.kt のコードレビュー
- `code-review` スキルを使用し、`MainActivity.kt` の内部ロジックを検査する。
- 括弧の整合性、状態フラグ漏れ、メモリリークを重点的に確認する。

## Step 2: UXフロー監査
- `ux-flow-audit` スキルを使用し、すべての `screen_*.xml` および `activity_main.xml` の構成を検査する。
- 全画面からの退出パスの有無、タイムアウト機構を総合診断する。

## Step 3: Temi SDK の利用状況監査
- `temi-sdk-check` スキルを使用し、プロジェクト内の Temi SDK 呼び出し箇所を検査する。
- 特に `goTo()` のステータスハンドリング、発話時のレイヤー制御を確認する。

## Step 4: API 30 互換性チェック
- Temi V3 は Android 11 (API Level 30) です。
- 全ての `res/drawable/*.xml` ファイルに対して、API 30 環境でクラッシュする原因となる要素（未対応の属性など）が含まれていないかチェックする。

## Step 5: 統合レポートの出力
- 各監査ステップの結果をまとめ、「重大」「中程度」「軽微」に分類した統合レポートを出力する。
- 出力フォーマットは必ず以下を含めること: `[ファイル名] : [行番号] - [問題内容] (修正案: ...)`

## Step 6: 自動修正の提案
- 抽出された「重大」クラスの問題に対して、ユーザーに自動修正の実施を提案し、承認を得次第修正を実行する。
