# UX Flow Audit Skill

アプリのUXフロー、画面遷移、およびユーザー体験（UX）に関する監査を実行する際に使用するスキルです。

## 検査項目

- **全退出パスの列挙**: 各画面（Screen）から遷移可能なすべての退出パスを列挙する。
- **デッドエンドの検出**: 「退出パスがない」状態は例外なく **【重大問題】** として報告する。
- **タイムアウト自動復帰の有無**: ユーザーの無操作状態が続いた場合に、初期画面（IDLE等）へ自動的に復帰するタイムアウト機構が全画面に存在するか確認する。
- **完全フロー検証**: 基本フロー `IDLE → GREETING → LISTENING → THINKING → GUIDANCE → NAVIGATION → ARRIVAL → IDLE` が途切れることなく循環できるか検証する。
- **異常パスのハンドリング**: 以下のような異常系パスが考慮され、リカバリーできているか確認する。
  - ネットワークエラー時のフロー
  - ASR（音声認識）の空結果時のフロー
  - ナビゲーションのAbort（中断/障害物）時のフロー
  - ユーザーによる明示的な中断操作時のフロー

## 出力形式

1. **テキスト版画面遷移図**: 現在実装されている各画面間の遷移関係をテキスト形式（矢印などを用いたフロー）で提示する。
2. **分析結果の指摘**:
   - **断点**: フローが途切れてしまう箇所（退出パスが存在しない画面）の指摘。
   - **無限ループ**: 予期せず同じ画面や処理をループしてしまう潜在的問題の指摘。
