# ユーザー体験(UX) 評価シート

本ドキュメントでは、Temi Guideアプリの現在の実装コードとUIレイアウトに基づき、客観的な観点からUXの評価と改善提案をまとめました。

## 評価結果（5段階評価）

| 評価項目 | スコア | 評価コメントと根拠 |
|---|:---:|---|
| **a. 第一印象** | 4.0 | 背景のアニメーション（Glow/Halo）と猫コンシェルジュのAVDアニメーションが動的で温かみがある。CTAボタンが大きく視覚的に明確。 |
| **b. 操作の直感性** | 4.5 | 音声入力を主体としており、必要以上のタップ操作を排除している。進行状況（プログレスバーやETA）が明瞭に可視化されている。 |
| **c. 応答速度** | 3.5 | Flash Liteモデルを利用しているためAIの応答は比較的速いが、生成〜TTS発話までの間 (THINKING画面) の体感時間がネットワークに依存する。 |
| **d. 案内の正確性** | 5.0 | Gemini APIとシステムプロンプトの強力な指示により、単語から適切なLocationへのマッピングが高度に行われている。 |
| **e. エラー時の対応** | 4.5 | タイムアウト、JSONパースエラー、ナビゲーションのAbortなど、ほぼすべての異常系からホームベースへの復帰パスが整備されている。 |
| **f. 会話の自然さ** | 4.0 | コーディネート提案などAIならではの返答は優秀。ただし、「戻って」などの割り込み系キーワードが固定文字列のマッチングに依存している点がある。 |
| **g. 視認性** | 4.0 | テキストサイズ（80sp等）が十分に大きく、ハイコントラストな配色が利用されている（例: `@color/brand_primary`）。 |
| **h. アクセシビリティ** | 4.0 | 英語・中国語によるマルチリンガル対応がシステムプロンプト層で解決されている点は非常に優れている。車椅子ユーザー等への画面高さ配慮は実機検証が必要。 |
| **i. 安心感** | 4.5 | 行き止まりがない（デッドエンド排除の徹底）。放置されても30秒で必ずホームに帰るため、人間側に「壊した」と思わせない設計。 |
| **j. 再利用意向** | 4.0 | スムーズなアニメーションと親しみやすい猫キャラクターにより、事務的な案内機ではなくアシスタントとして好感を持てる。 |

**総合スコア: 4.2 / 5.0**

## 現状の課題と優先改善リスト

コードとレイアウトの構造から推察される、今後のプロダクト化に向けた優先度の高い改善項目です。

### 🔴 High Priority (必須改善)
1. **割り込み音声の認識精度向上**: 現在の実装では、音声認識(ASR)の入力確定までアクションが起こりません。ユーザーが「待って！」「ストップ！」と言った瞬間に即座に停止できる割り込み(Barge-in)対応をTemi SDKのStream処理等を利用して強化すると、安全性が飛躍的に向上します。
2. **ネットワーク遅延のUXカバー**: `screen_thinking.xml` が表示されている間、APIの応答が遅れた場合にユーザーが「フリーズした」と誤認する可能性があります。プロンプト（例: 「考えています…」）だけでなく、数秒経過したら「AIに確認中です、もう少々お待ち下さい」と発声するフェイルセーフを追加すべきです。

### 🟡 Medium Priority (推奨改善)
1. **多言語のUI表記**: 音声対話はAIによって多言語化されていますが、XML上の固定テキスト（例: `string/guidance_title` や `string/staff_title`）は Androidの `values-en/`, `values-zh/` を用いた Locale 連動、もしくは AIの言語指定に基づく動的テキストへの切り替えが必要です。
2. **地図ビューの表示**: ナビゲーション中(`screen_navigation.xml`)にプログレスバーだけでなく、簡易的な店内ミニマップ画像を表示することで、「どこに向かっているか」の空間把握を助け安心感を与えられます。

### 🟢 Low Priority (追加機能)
1. **スタッフ呼び出しの連携**: 現在の `call_staff` は30秒間の「見せかけのアニメーション待機」です。本番化にあたり、バックグラウンドのWebhook等を利用して実際のスタッフのインカムや店員端末へPINGを送信する実装へ置き換える必要があります。
